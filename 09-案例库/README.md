# 第9章 案例库

> 📍 所属：第9章 案例库 · 层级：导航 · 本章是知识库的"证据层"，沉淀真实业务案例，说明案例库的定位、目录组织、命名规范、入库标准与标签体系

## 一、定位

02~06 章回答"一般该怎么做"，案例库回答"具体某一次是怎么做的、结果如何"。三个用途：

1. **决策参考**：动手前搜同类案例，看看前人在相似情境下选了什么、结果怎样；
2. **经验固化**：把项目复盘从"聊完就散"变成可检索的资产，对抗遗忘与记忆美化；
3. **知识校验**：方法论章节的说法由案例佐证或修正——同一结论在案例中反复出现，就上升进方法论章节；反复被打脸，就修订原章节。

所有案例统一使用四段结构：**背景 → 动作 → 结果 → 可迁移经验**。写法详见 [8.3 案例库建设指南](../08-学习路径与资源/8.3-案例库建设指南.md)，直接复制 [案例模板](./案例模板.md) 开写。

## 二、目录组织方式

按领域分五个子目录，与 02~06 章的领域划分对齐。本 README 先约定用途，子目录在收到第一个案例时随案例创建即可（Git 不跟踪空目录）：

| 子目录 | 收录什么 | 对应章节 |
| --- | --- | --- |
| `产品/` | 需求决策、功能设计、PRD 落地、版本迭代、砍需求等案例 | [02-产品](../02-产品/README.md) |
| `运营/` | 活动运营、内容运营、用户运营、渠道投放、社群等案例 | [03-运营](../03-运营/README.md) |
| `数据分析/` | 专项分析、指标体系搭建、A/B 测试、归因、异动排查等案例 | [04-数据分析](../04-数据分析/README.md) |
| `商业分析/` | 商业模式测算、竞争分析、定价、市场进入/退出决策等案例 | [05-商业分析](../05-商业分析/README.md) |
| `增长/` | 跨产品与运营的增长实验、裂变、留存战役、AARRR 全链路等案例 | [06-交叉领域/6.1-增长](../06-交叉领域/6.1-增长.md) |

**归属判断**：按案例的**主要决策属于哪个职能**归类，只放一处（MECE 原则，见 [0.2 知识管理方式](../00-使用说明/0.2-知识管理方式.md)）。跨领域案例的典型裁决：A/B 测试验证功能 → 若核心看点是实验设计放 `数据分析/`，若核心看点是功能决策放 `产品/`；拉新活动 → 单次活动放 `运营/`，成体系的实验循环放 `增长/`。拿不准时放主导者所属的领域。

## 三、命名规范

文件名格式：**`YYYYMM-案例主题.md`**

- `YYYYMM`：案例所述事件的发生年月（不是写作年月），保证目录内按时间自然排序；
- `案例主题`：8-20 字，写"对象 + 动作/事件"，让人不点开就知道讲什么。

示例：

```
09-案例库/
├── README.md
├── 案例模板.md
├── 运营/
│   ├── 202603-签到功能改版提升次日留存.md
│   └── 202605-618大促推送时段AB测试.md
├── 产品/
│   └── 202604-砍掉语音评论需求的决策复盘.md
└── 增长/
    └── 202601-邀请裂变活动ROI不达预期复盘.md
```

反例：`案例1.md`（无信息量）、`关于提升留存的一些尝试.md`（无日期、主题含糊）、`202603签到.md`（缺连字符、主题过短）。

## 四、入库标准

满足以下**任意一条**即值得入库（详细展开见 [8.3 案例库建设指南](../08-学习路径与资源/8.3-案例库建设指南.md)）：

1. **反直觉**：结果与事前预期明显不符；
2. **有取舍**：经历了真实的方案选择，且取舍逻辑可复用；
3. **有教训**：踩了现有误区章节没覆盖、或覆盖了但没能识别的坑。

不收：纯执行流水账、结果无法验证的、归因说不清的。质量优先于数量——**宁缺毋滥是案例库的生命线**。

## 五、标签体系

每个案例在元信息区打一行标签，格式：`标签：#领域 #主题1 #主题2 #类型`。

### 5.1 三个维度

| 维度 | 规则 | 词表 |
| --- | --- | --- |
| 领域（必选 1 个） | 与所在子目录一致 | `#产品` `#运营` `#数据分析` `#商业分析` `#增长` |
| 主题（选 2-4 个） | 优先复用下方词表，新词按需补充并更新本表 | `#留存` `#转化率` `#拉新` `#召回` `#活动` `#推送` `#AB测试` `#漏斗` `#归因` `#指标体系` `#定价` `#商业化` `#竞品` `#冷启动` `#裂变` `#PRD` `#需求决策` |
| 类型（必选 1 个） | 案例性质 | `#成功` `#失败` `#反直觉` `#决策复盘` |

### 5.2 维护规则

1. 主题词表以本 README 为唯一权威版本：新增主题标签时，先检查是否有近义词已在表中，确需新增则同时更新上表；
2. 标签写法统一（如统一用 `#AB测试`，不用 `#ab实验`），发现不一致在每周复盘时归并；
3. 常用检索命令：

```bash
# 列出全库标签使用频次
grep -rh "^标签：" 09-案例库/ | tr ' ' '\n' | grep '^#' | sort | uniq -c | sort -rn

# 找出所有"失败"的"留存"类案例
grep -rl "#留存" 09-案例库/ | xargs grep -l "#失败"
```

## 六、写作与维护流程

1. **写**：复制 [案例模板](./案例模板.md) 到对应子目录，按命名规范重命名，30-45 分钟完成初稿；
2. **连**：在案例的"关联知识点"处链接对应章节，并到该章节的"关联阅读"补回链（双链约定见 [0.2](../00-使用说明/0.2-知识管理方式.md)）；
3. **养**：每周复盘时筛选新案例、回填"待补"数据；每季度横向重读，把重复出现的结论上升进方法论章节。整套节奏见 [8.3 案例库建设指南](../08-学习路径与资源/8.3-案例库建设指南.md) 第四节。
