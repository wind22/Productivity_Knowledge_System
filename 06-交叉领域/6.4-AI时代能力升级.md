# 6.4 AI 时代能力升级

> 📍 所属：第6章 交叉领域专题 · 交叉领域：产品 × 运营 × 数据分析 × 商业分析 × AI · 一句话说明：AI（尤其是大语言模型）如何重构分析工作流、催生 AI 产品的新指标体系、并重写内容运营的生产流水线

## 1. AI 辅助分析：把 LLM 变成你的分析副驾

### 1.1 是什么/为什么

大语言模型（LLM）不会替代分析师，但**会用 LLM 的分析师会替代不会用的分析师**。LLM 在分析工作中的正确定位是"副驾"：擅长解读、起草、聚合、翻译（数据→语言），不擅长精确计算和事实记忆。分工原则：

```
计算与取数 → SQL/Python/BI 工具（确定性）
解读、起草、聚合 → LLM（生成性）
判断、担责、拍板 → 人（问责性）
```

**使用红线**：喂给 LLM 的数据注意脱敏与公司数据安全政策；LLM 输出的所有数字必须回到原始数据核对（LLM 会一本正经地算错数、编数字）。

### 1.2 场景一：数据解读（How + 提示词思路）

把跑好的数据表贴给 LLM，让它做"异动扫描 + 假设生成"，人再去验证假设。

**提示词骨架（可直接改用）**：

```
角色：你是一名电商行业资深数据分析师。
背景：这是我们 App 过去 8 周的分渠道留存数据（表格如下）。
      业务背景：6 月 15 日改版了新用户引导流程；6 月下旬有竞对大促。
任务：
1. 找出数据中最显著的 3 个异动（说明判断依据：环比幅度/趋势拐点/组间差异）；
2. 对每个异动给出 2-3 个候选原因假设，并注明"验证该假设需要看什么数据"；
3. 明确指出哪些结论置信度低、需要人工复核。
约束：不要编造表中不存在的数字；区分"数据显示"与"我的推测"。
```

要点：**给业务上下文**（改版、大促等事件）是 LLM 解读质量的最大变量；**要求它自报置信度和验证路径**，把它的输出从"结论"降级为"待验证假设清单"——这正是分析师最耗时的头脑风暴环节。

### 1.3 场景二：报告初稿（How + 提示词思路）

**提示词骨架**：

```
角色：你是向 CEO 汇报的商业分析师，写作遵循金字塔原理、结论先行。
输入：以下是我的分析结论要点（人工整理的 bullet，含关键数字）+ 目标读者 + 汇报时长。
任务：生成周报/月报初稿，结构为：一句话总结 → 3 个关键发现（每个发现=结论+数据支撑+
      建议动作）→ 风险提示 → 下一步。
风格：每个小标题直接写结论（不写"XX 分析"）；全文不超过 600 字；数字保留我给出的原值。
```

关键做法：**人给结论要点，LLM 负责组织语言**——顺序反过来（让 LLM 从原始数据直接生成报告）就会产出幻觉数字和平庸结论。迭代技巧：让它先出 3 个不同角度的提纲，人选一个再展开，比直接生成全文质量高得多。

### 1.4 场景三：竞对情报聚合（How + 提示词思路）

**提示词骨架**：

```
任务：以下是竞对 X 最近的公开材料（财报电话会纪要/发布会文字稿/应用商店版本更新
      日志/招聘 JD，粘贴于下）。请：
1. 按"产品动作 / 商业化动作 / 组织信号"三类归纳事实（每条标注来源材料）；
2. 推断其未来 1-2 个季度的战略重心，列出推理链条；
3. 指出对我方（背景：____）威胁最大的 1 件事与可借鉴的 1 件事。
约束：事实与推断分开呈现；没有依据的不要写。
```

高价值信息源：招聘 JD（组织投入方向）、版本更新日志（产品节奏）、财报电话会（管理层原话）。LLM 的价值在于把几万字的非结构化材料压缩成结构化情报，**人的价值在于选料和交叉验证**。

### 1.5 通用提示词心法

| 心法 | 说明 |
|------|------|
| 给角色和读者 | "你是 X，写给 Y 看" 决定了输出的深度和口吻 |
| 给上下文 | 业务背景、时间事件、口径定义——上下文质量决定输出上限 |
| 给结构 | 明确输出格式（表格/分节/字数），避免自由发挥 |
| 要求自报不确定性 | "区分事实与推测""标注置信度" 是防幻觉的第一道闸 |
| 迭代而非一次成型 | 先提纲后正文、先批判后修改（"指出这份初稿的 3 个弱点再重写"） |

## 2. AI 产品的新指标体系

### 2.1 为什么传统指标不够用

传统互联网产品"边际成本≈0"的假设在 AI 产品上失效：**每次调用都有真金白银的推理成本**；且产出质量不确定（同一输入可能好可能坏），"功能是否可用"变成了概率问题。因此指标体系要在传统 DAU/留存之外新增三组。

### 2.2 成本指标：token 经济学

```
单次任务成本 = (输入 token 数 × 输入单价 + 输出 token 数 × 输出单价) × 平均重试次数
毛利视角：单用户毛利 = ARPU − 单用户月均任务数 × 单次任务成本 − 其他变动成本
```

| 指标 | 含义 | 优化手段 |
|------|------|----------|
| 每任务平均 token 消耗 | 成本的最小核算单元 | 提示词精简、上下文裁剪、缓存命中（KV Cache/前缀缓存） |
| 模型分层调用率 | 高价大模型 vs 低价小模型的调用占比 | 路由策略：简单任务走小模型，复杂任务升级大模型 |
| 缓存命中率 | 重复请求直接返回缓存的比例 | 高频问题缓存、embedding 相似问合并 |
| 免费用户成本上限 | 免费额度设计的财务护栏 | 免费层限次数/限模型，防止"用户越活跃亏得越多" |

**示例**：Kimi、豆包等 C 端助手在爆发期都经历过"DAU 涨 = 亏损涨"的阶段，因此普遍采用小模型预路由 + 深度思考模式需手动触发/限额的设计——这本质上是把"成本指标"做进了产品交互。

### 2.3 质量指标：生成质量评估

生成质量无法像崩溃率那样直接采集，需要分层评估体系：

1. **自动化离线评估**：建立黄金评测集（覆盖核心场景的输入+理想输出），每次模型/提示词变更前跑分。客观任务用精确指标（准确率、格式合规率），主观任务用 **LLM-as-a-Judge**（用更强的模型按 rubric 给生成结果打分），并定期用人工标注校准 Judge 的偏差。
2. **在线隐式信号**：采纳率（用户接受生成结果的比例）、编辑距离（用户改了多少才使用）、重新生成率、复制/下载率、对话轮次中断位置。**采纳率和编辑距离是 AI 产品最重要的隐式质量指标**——相当于传统产品的转化率。
3. **在线显式反馈**：点赞点踩、举报率、幻觉投诉率（需要客服/审核通道回流打标）。
4. **安全底线指标**：有害内容拦截率、越狱成功率、敏感话题误伤率（合规红线，单独监控）。

**示例**：GitHub Copilot 公开的核心指标是代码建议采纳率与保留率（采纳的代码在 N 分钟后仍存在的比例）——后者比采纳率更能反映真实质量。国内如通义灵码、MarsCode 也采用类似口径。

### 2.4 效率指标：人机协作

AI 产品的终极价值主张是"帮人省时间/提产出"，需要直接度量协作效率：

| 指标 | 定义 | 示例 |
|------|------|------|
| 任务完成时间对比 | 用 AI vs 不用 AI 完成同类任务的时长 | 客服辅助场景：均单处理时长 −30% |
| 人工介入率 | 需要人接管/纠错的任务占比 | AI 客服的转人工率 |
| 一次通过率 | 无需重试/修改即可用的比例 | 文生图的一次出图可用率 |
| 信任校准度 | 用户采纳错误输出（过度信任）与拒绝正确输出（信任不足）的比例 | 医疗/金融等高风险场景必看 |
| 技能留存 | 长期使用后用户独立完成能力是否退化 | 教育类 AI 产品的护栏指标 |

## 3. AIGC 对内容运营生产流水线的重构

### 3.1 是什么/为什么

传统内容运营流水线：`选题 → 创作 → 审核 → 分发 → 复盘`，瓶颈在"创作"环节（人力密集、产能线性）。AIGC 把瓶颈整体后移：**创作产能近乎无限，瓶颈变成选题判断、质量审核与账号/渠道承载力**。这不是单点提效，而是流水线的重构。

### 3.2 新流水线各环节（How）

**（1）选题：从灵感驱动到数据驱动**
- 用 LLM 聚合热榜、评论区、搜索词，批量生成候选选题 + 预估角度；
- 人的新角色：从"想选题"变成"从 100 个候选里挑 10 个 + 判断品牌契合度"。
- 示例：美妆品牌运营把小红书热门笔记标题批量喂给 LLM 做"爆款要素拆解"（人群×场景×痛点×钩子结构），再组合生成本品牌的选题矩阵。

**（2）创作：一鱼多吃的批量生成**
- 核心资产从"单篇内容"变为"内容母版 + 提示词模板库"：一篇深度内容（母版）由 AI 派生出公众号长文、小红书笔记（3 种人设口吻）、抖音口播稿、社群话术、评论区互动脚本；
- 图文：文案 LLM + 配图文生图模型；视频：脚本生成 + 数字人口播/AI 剪辑，直播切片自动化；
- 建立**品牌语料库**（历史优质内容、品牌禁忌词、口吻范例）作为提示词的固定上下文，保证批量生成不跑偏。

**（3）审核：从抽查到全量拦截 + 人工终审**
- AIGC 时代审核压力倍增（产量 ×10）且风险类型新增：幻觉事实错误、AI 味（同质化腔调）、版权与平台标识合规（多平台已要求 AI 生成内容显著标识）；
- 三道闸：机器审核（敏感词/竞品词/法律禁语）→ LLM 自检（"检查这篇稿件中的事实性声明并列出待核实项"）→ 人工终审（品牌调性与事实抽核）。**人工终审不可省**——AI 生成事故（编造数据、翻车文案）对品牌的伤害远大于省下的人力。

**（4）分发：千人千面的规模化 A/B**
- 同一内容自动生成 5-10 个标题/封面变体投放测试，用数据回收最优版本再放量；
- 私域场景：按用户分层批量生成个性化触达文案（新客/沉默/高价值话术不同），把过去只有头部客户享受的 1v1 运营下放到全量用户。

**（5）复盘：数据回流反哺提示词**
- 把高互动内容回流进语料库、把翻车案例写进负面清单，提示词模板库像代码一样做版本管理与迭代——**提示词模板库成为内容团队的核心资产与护城河**。

### 3.3 组织与能力的变化

| 维度 | AIGC 之前 | AIGC 之后 |
|------|-----------|-----------|
| 团队结构 | 按内容形态分工（图文组/视频组） | 按环节分工：选题策划 + 提示词工程/内容中台 + 审核质检 |
| 核心能力 | 文笔、网感 | 判断力（选题与审美）、提示词工程、数据复盘 |
| 产能模型 | 人数 × 人均篇数（线性） | 母版数 × 派生系数 × 通过率（杠杆型） |
| 关键指标 | 发布量、单篇互动 | 单位有效内容成本、AI 内容一次通过率、内容矩阵总 ROI |

**示例**：B站与抖音的大量知识区/资讯类账号已采用"AI 初稿 + 真人出镜/配音精修"的半自动流水线，周更变日更；电商平台商家用 AIGC 批量生成商品图（换背景/换模特）与详情页文案，制作成本降一个数量级——竞争焦点随之从"能不能做出内容"转移到"选题判断与品牌一致性"。

### 3.4 落地检查清单

- [ ] 是否建立了品牌语料库与禁忌清单，并接入所有生成流程？
- [ ] 提示词模板是否有版本管理和 Owner？
- [ ] AI 生成内容是否按平台要求做了标识与合规审核？
- [ ] 是否监控"AI 味"同质化（互动率衰减）而不仅是产量？
- [ ] 人工终审环节是否保留且有明确的抽检比例？
- [ ] 是否用"单位有效内容成本"而非"产量"考核流水线？
