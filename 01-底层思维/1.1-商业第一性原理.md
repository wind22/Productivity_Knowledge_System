# 1.1 商业第一性原理

> 📍 所属：第1章 底层思维层 · 本文回答"商业到底是怎么运转的"——所有产品、运营、商业化动作，最终都要通过这几条底层公式来检验。

## 一、商业的本质：创造价值 → 传递价值 → 捕获价值

### 是什么

任何可持续的商业，都由三个环节构成一条价值链：

| 环节 | 含义 | 对应的互联网职能 |
|------|------|----------------|
| **创造价值** | 做出比现有方案更好地满足需求的东西 | 产品、研发、供给运营 |
| **传递价值** | 让目标用户知道、获得、用起来这个东西 | 市场、渠道、增长、用户运营 |
| **捕获价值** | 从创造的总价值中切出一部分变成收入 | 商业化、销售、变现策略 |

为什么这个三段论重要：**三个环节缺一不可，且顺序不能颠倒**。很多失败的项目死因可以归类为：
- 只有传递没有创造 → 疯狂投放但产品不行，买来的用户全流失（大量烧钱补贴的 O2O 项目）；
- 只有创造没有传递 → "好产品自己会说话"的幻觉，酒香也怕巷子深；
- 捕获过度侵蚀创造 → 广告加载率过高伤害体验，杀鸡取卵。

### 如何应用

拿到任何一个业务问题，先定位它属于哪个环节，再用对应领域的方法解决：

1. 用户增长慢 → 先判断是"创造"问题（产品没价值，激活/留存差）还是"传递"问题（渠道没打开，获客贵）。**留存差时做投放是烧钱，激活好时不投放是浪费**。
2. 收入不达标 → 判断是价值总盘子不够大（创造环节），还是变现率太低（捕获环节）。
3. 做年度规划时，检查三环节资源配比是否失衡。

### 示例：抖音的三段论

- 创造价值：推荐算法 + 低创作门槛，让"杀时间"这件事的体验远超以往（价值来源是用户时长与愉悦感，以及创作者的流量与收入）；
- 传递价值：早期靠明星运营、挑战赛、春晚级投放快速破圈；
- 捕获价值：在用户价值稳固后，才逐步加信息流广告、直播打赏、电商闭环——变现节奏严格服从于价值创造的成熟度。

## 二、用户价值公式（俞军公式）

### 是什么

前百度贴吧、滴滴产品负责人俞军提出：

```
用户价值 = (新体验 − 旧体验) − 替换成本
```

- **新体验**：你的产品给用户的体验总量；
- **旧体验**：用户当前解决方案的体验（注意：竞品、线下方式、甚至"不解决"都是旧体验）；
- **替换成本**：学习成本、迁移成本（数据/关系链）、金钱成本、心理风险。

为什么重要：它解释了"产品比竞品好一点为什么没用"——**只要 (新−旧) 的差值小于替换成本，用户就不会迁移**。这是无数"更好用的微信替代品"全部失败的根本原因：微信的旧体验本身很高，而关系链带来的替换成本近乎无穷大。

### 如何应用

| 提升杠杆 | 手段 | 示例 |
|---------|------|------|
| 提高新体验 | 打磨核心功能、性能、内容质量 | 抖音把"打开即播放"做到极致 |
| 找旧体验差的人群 | 选择被现有方案服务得最差的用户切入 | 拼多多切入五环外、对价格极度敏感而旧电商体验差的人群 |
| 降低替换成本 | 一键导入、免注册试用、新人补贴 | 支付类产品的首绑立减；游戏的"一键继承好友关系" |

实操检查清单（做需求评审 / 竞品分析时）：
- [ ] 用户当前的旧体验是什么？给它打个分；
- [ ] 我们的新体验相对旧体验的提升，是 10% 还是 10 倍？（10% 的提升通常抵不过替换成本）
- [ ] 替换成本清单：金钱 / 学习 / 数据迁移 / 关系链 / 心理风险，哪项最大？能否由我们代付？

### 示例：拼多多 vs 淘宝

拼多多早期对目标用户：新体验 =极低价格 + 微信内即拼即买（免下载、免复杂搜索）；旧体验 = 淘宝对下沉用户其实并不友好（选择过载、需要独立 App、价格不够低）；替换成本 ≈ 0（微信小程序 + 免注册 + 拼团社交裂变代付了获客与信任成本）。三项相乘，用户价值巨大——这不是"消费降级"，而是精确的公式套利。

## 三、企业价值公式：利润拆解

### 是什么

```
利润 = 收入 − 成本
     = 用户数 × ARPU − (固定成本 + 变动成本)
```

- **ARPU**（Average Revenue Per User）：单用户平均收入，可继续拆为 `付费率 × ARPPU`；
- **固定成本**：不随用户量变化（研发、办公、服务器保底）；
- **变动成本**：随用户/订单量线性增长（获客补贴、带宽、履约、客服）。

为什么重要：互联网商业模式的核心秘密就藏在成本结构里——**软件的边际成本趋近于零**，所以固定成本被巨大用户量摊薄后，利润率可以极高（微信、王者荣耀）；而外卖、打车这类含线下履约的业务，变动成本随单量线性增长，必须精算单均经济模型（Unit Economics）。

### 如何应用

1. **给任何业务算一笔 Unit Economics**：`单均毛利 = 单均收入 − 单均变动成本`；若单均毛利为负且没有规模化转正的路径，增长越快亏得越多。
2. **判断增长质量**：核心指标 `LTV > CAC`（用户生命周期价值 > 获客成本），通常要求 LTV/CAC > 3 才算健康。
3. **找利润杠杆**：利润公式的每个变量都是一个可运营的抓手——拉用户数（增长）、拉付费率（转化运营）、拉 ARPPU（会员分层/客单价）、降变动成本（供应链/调度效率）。

### 示例：美团外卖的单均模型

美团外卖每单收入 ≈ 佣金 + 配送费 + 广告分摊；每单变动成本大头是骑手配送成本。早期每单亏损，靠三件事转正：订单密度提升 → 骑手单均配送成本下降（一趟顺路送三单）；广告变现打开 → 单均收入提升；会员/拼好饭分层 → 不同价格敏感度用户被不同成本结构服务。**这就是"用公式经营"：每个季度财报里的单均盈利改善，都能映射回公式里的某个变量。**

## 四、交易成本理论（科斯）：平台为什么存在

### 是什么

经济学家科斯（Ronald Coase）提出：市场交易不是免费的，存在**交易成本**——搜寻与信息成本、比较与议价成本、决策成本、执行与监督成本、违约救济成本。企业（以及平台）之所以存在，是因为它们能把某些交易的成本降到比自由市场更低。

推论：**互联网平台的本质，就是用技术系统性地降低某类交易成本**。哪里交易成本高，哪里就有平台机会。

### 如何应用

分析或设计任何平台型业务时，逐项检查它降低了哪类交易成本：

| 交易成本 | 平台的降低手段 | 示例 |
|---------|--------------|------|
| 搜寻成本 | 搜索、推荐、分类 | 淘宝搜索/千人千面；抖音推荐 |
| 信息不对称 | 评价体系、认证、图文详情 | 大众点评评分；淘宝店铺信用 |
| 议价成本 | 明码标价、平台定价、竞价机制 | 滴滴统一计价（替代路边议价） |
| 信任/履约成本 | 担保交易、保险、售后规则 | 支付宝担保交易（淘宝起飞的真正引擎） |
| 匹配/调度成本 | 算法实时匹配 | 滴滴派单、美团骑手调度 |

反向应用：如果你的"平台"没有显著降低任何一项交易成本，它就只是个信息展示网站，抽不到佣金——**平台抽成的合法性上限 = 它降低的交易成本总量**。抽成超过这个量，供需双方就会"飞单"（绕过平台私下交易），这解释了为什么家政、家教类平台普遍难做。

### 示例：支付宝的诞生

2003 年淘宝最大的交易成本是**信任**：买家怕付了钱不发货，卖家怕发了货收不到钱。支付宝的担保交易（买家确认收货后平台才打款给卖家）直接消灭了这项成本，网购渗透率从此起飞。先有交易成本的洞察，才有支付工具——顺序不能反。

## 五、供需关系：一切增长与商业化问题的最终归宿

### 是什么

供需是经济学的地基：价格与量由供给曲线和需求曲线的交点决定。对互联网业务的翻译：

- **需求侧**：有多少用户、带着多强的意愿和支付能力来；
- **供给侧**：有多少商品/内容/服务/运力可被消费，质量和成本如何；
- **匹配效率**：平台把供需撮合起来的速度与精度。

为什么它是"最终归宿"：增长问题拆到最后，要么是需求不足（没人来），要么是供给不足（来了没东西可消费），要么是匹配不行（有人有货但对不上）；商业化问题（定价、补贴、抽成）本质是**在供需曲线上移动交点**。

### 如何应用

1. **诊断增长瓶颈先分侧**：DAU 不涨，先看是拉新问题还是"来了留不住"；留不住再看是不是供给问题——很多内容社区的留存差根因是"垂类内容供给不足"，砸钱拉新无解，正解是补供给。
2. **补贴要补短板侧**：双边平台冷启动，把钱花在更稀缺的一侧。滴滴早期重补司机（供给稀缺），司机密度上来后接单快，需求自然增长。
3. **警惕供给同质化**：供给严重同质化时，平台内卷、价格战、商家利润归零（外卖商家困境）；差异化供给才有定价权。
4. **动态定价 = 用价格调节实时供需**：打车高峰溢价不是"宰客"，而是用价格召回供给（司机出车）并抑制非刚性需求。

### 示例：B 站的供需运营

B 站的核心资产是 UP 主（供给）。它的关键动作长期围绕供给侧：创作激励计划、充电/直播分成、垂类扶持（知识区、游戏区）——因为对内容平台而言，**优质供给自带需求**（用户为特定 UP 主而来），而纯买量拉来的需求在供给不足时会迅速流失。判断一家内容/电商平台的健康度，看它供给侧的增长与分层，比看 DAU 更本质。

## 六、五个原理的合并使用：一张分析底图

拿到任何商业问题，按此顺序自检：

1. 这门生意**创造**了什么价值？（用户价值公式：新−旧−替换成本 是否为正且够大）
2. 价值如何**传递**？供需两侧谁是瓶颈？匹配效率如何？
3. 平台/企业降低了哪类**交易成本**？抽成/变现是否超过了它降低的成本量？
4. **利润公式**里每个变量的现状与杠杆是什么？Unit Economics 能否转正？
5. 以上结论在竞争与时间维度下是否可持续？（引出 1.2 的博弈思维与二阶思维）
