# 第1章 底层思维层

> 📍 所属：第1章 底层思维层 · 本章是产品、运营、数据分析、商业分析四个领域**共用的第一性原理层**，解决"为什么这些方法有效"的问题。

## 本章定位

在这个知识库中，02-05 章分别讲产品、运营、数据分析、商业分析的具体方法论。但如果只学"术"不学"道"，会陷入两个困境：

1. **方法失灵时不知所措**：AARRR 模型套不上业务时，你需要回到"供需匹配"这个更底层的框架重新思考；
2. **跨领域迁移困难**：做产品的人转做商业分析，如果只会画原型，等于从零开始；但如果掌握了用户价值公式和交易成本理论，很多判断是相通的。

本章把四个领域背后共同依赖的底层原理抽出来单独成章：**商业原理、心智模型、决策框架、用户心理学**。无论你之后读哪一章，遇到"为什么"的问题，都可以回到这里找答案。

## 与其他章节的关系

```
01 底层思维（道）
 ├─→ 02 产品：用户价值公式 → 需求判断；用户心理学 → 产品设计
 ├─→ 03 运营：Fogg/Hook 模型 → 用户激励；复利思维 → 留存运营
 ├─→ 04 数据分析：概率思维 → 统计推断；5 Why/议题树 → 归因分析
 ├─→ 05 商业分析：交易成本/供需 → 商业模式分析；博弈思维 → 竞争分析
 └─→ 06 交叉领域：系统思维+杠杆思维 → 增长模型与飞轮设计
```

## 建议阅读方式

- **首次通读**：按 1.1 → 1.2 → 1.3 → 1.4 顺序，约 2-3 小时；
- **按需查阅**：做优先级排序时查 1.3 的 ICE/RICE；做转化优化时查 1.4 的行为模型；
- **回炉重读**：每当具体方法论"不好使"时，回到本章找更底层的解释。
