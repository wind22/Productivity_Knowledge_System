> 覆盖领域：**产品 × 运营 × 数据分析 × 商业分析**

## 🧭 知识分层模型

本库所有内容按五层组织（详见 [0.1 知识分层模型](00-使用说明/0.1-知识分层模型.md)）：

```
原理层 (Why)   —— 事物的本质与第一性原理，最稳定，最少变
概念层 (What)  —— 领域核心概念与术语，构成思考的词汇表
方法层 (How)   —— 可复用的方法论与流程框架
技巧层 (Hack)  —— 实战技巧与经验法则，见效快但依赖场景
案例层 (Case)  —— 真实案例沉淀，验证与修正上面四层
```

## 🔎 双入口索引

1. **按领域索引**：即本页目录，适合系统学习、建立领域知识框架。
2. **按问题场景索引**：见 [INDEX-场景索引](INDEX-场景索引.md)，适合带着具体业务问题（如"如何提升留存""怎么定价"）快速定位相关知识点。

## 🔄 维护约定

- **每周复盘沉淀**：把一周内的新认知、踩坑、案例写入对应章节或案例库（SOP 见 [0.3 更新机制](00-使用说明/0.3-更新机制.md)）。
- **案例入库标准**：有明确的动作与结果数据、有可迁移经验的才入库。
- **过期知识淘汰**：不再适用的内容在文档头部标注 `⚠️ 已过时`，并注明原因与替代结论，保留供追溯。
