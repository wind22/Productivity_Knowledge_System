# 00-使用说明

> 📍 所属：第0章 · 本章是整个知识库的"操作手册"：知识按什么结构组织、用什么方法管理、如何持续更新、怎么快速检索。第一次使用本库，请先读完本章（约 20 分钟）。

## 本章解决什么问题

一个知识库能否长期有用，取决于四件事是否想清楚：**放哪（结构）→ 怎么放（方法）→ 怎么保鲜（更新）→ 怎么找（索引）**。本章四篇文档分别回答这四个问题：

| 文档 | 回答的问题 | 什么时候读 |
|------|-----------|-----------|
| [0.1-知识分层模型](0.1-知识分层模型.md) | 一条知识该放到哪一层？为什么各领域章都是"原理→概念→方法论→技巧→误区"五件套？ | 首次使用；每次入库拿不准位置时 |
| [0.2-知识管理方式](0.2-知识管理方式.md) | MECE 分类、双链笔记、渐进式总结三种方法是什么？在纯 Markdown + Git 环境下怎么落地？ | 首次使用；想优化个人笔记流程时 |
| [0.3-更新机制](0.3-更新机制.md) | 每周复盘怎么做？案例达到什么标准才入库？过时知识如何标记和淘汰？ | 开始维护本库前必读；每周复盘时对照 |
| [0.4-索引方式](0.4-索引方式.md) | 系统学习和带着问题查库，分别走哪个入口？场景索引如何维护？ | 首次使用；找东西超过 5 分钟没找到时 |
| [0.5-概念权威页对照表](0.5-概念权威页对照表.md) | 跨章概念（RFM、LTV/CAC、A/B 测试…）以哪一页为准？更新时改哪里？ | 修订跨章概念前必查；新增概念时登记 |

## 四篇文档的关系

```
0.1 分层模型 ──→ 决定知识的"纵向坐标"（Why/What/How/Hack/Case）
0.2 管理方式 ──→ 决定知识如何入库、连接、持续精炼
0.3 更新机制 ──→ 让上面两条规则每周真实运转、让过期内容退出
0.4 索引方式 ──→ 让存进去的知识能在 5 分钟内被找到
0.5 权威页表 ──→ 保证同一概念全库只有一个"事实来源"，防止口径漂移
```

## 快速上手（3 步）

1. **只想找答案**：直接打开根目录 [INDEX-场景索引](../INDEX-场景索引.md)，按问题场景（如"如何提升留存"）进入，用法见 [0.4-索引方式](0.4-索引方式.md)；
2. **想系统学习**：从各领域章的 README 进入（[产品](../02-产品/README.md) / [运营](../03-运营/README.md) / [数据分析](../04-数据分析/README.md) / [商业分析](../05-商业分析/README.md)），按 x.1→x.5 顺序阅读，配合 [8.1-分阶段学习路径](../08-学习路径与资源/8.1-分阶段学习路径.md)；
3. **想往库里存东西**：先读 [0.1](0.1-知识分层模型.md) 确定坐标，按 [0.2](0.2-知识管理方式.md) 的规则写入并补链接，按 [0.3](0.3-更新机制.md) 的 Git 约定提交。
