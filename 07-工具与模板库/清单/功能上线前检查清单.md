# 功能上线前检查清单

> 使用说明：复制本清单到需求的上线工单/文档中，逐项勾选。**每一项都要有人名和结论**，"应该没问题"不算完成。建议上线前一天由产品牵头，产研测数一起过一遍；不适用的项标注「N/A + 原因」，不允许直接删除。
>
> 适用范围：面向用户的功能迭代。纯后端重构可裁剪「需求与验收」部分，但灰度与回滚部分不可裁剪。

---

## 一、需求与验收

- [ ] **PRD 终版已归档**：以评审通过的版本为准，口头改动已回写文档，避免"上线的和评审的不是一个东西”
- [ ] **验收标准逐条通过**：PRD 第 8 节验收标准（AC）逐条有测试结论，全部通过或有明确的带病上线决策记录
- [ ] **产品走查完成**：产品经理在测试环境亲自走完主流程与关键异常分支，不只看测试报告
- [ ] **需求范围确认**：本期「不做」的部分没有被顺手做进去，也没有 P0 功能被悄悄砍掉未同步
- [ ] **法务/合规确认**（涉及用户协议、隐私、资金、内容时）：合规意见已落实并留档

## 二、数据埋点

- [ ] **埋点逐个验证**：PRD 事件表中每个事件在测试环境实际触发并在埋点平台查到数据，属性值正确（不是"代码已写"）
- [ ] **触发时机核对**：曝光类事件确认是"渲染完成"还是"接口请求"触发，与口径文档一致
- [ ] **核心报表就绪**：上线后要看的看板/漏斗已建好，能当天出数，而不是上线后再补
- [ ] **旧埋点兼容**：改动是否影响存量事件（改名/删除/含义变化）？受影响的下游报表负责人已通知

## 三、兼容与性能

- [ ] **机型与系统覆盖**：Top 机型 + 最低支持系统版本实测通过；低端机无明显卡顿
- [ ] **旧版本兼容**：不升级 App 的老版本用户行为符合预期（降级展示或无感知），接口对旧版本向后兼容
- [ ] **弱网与断网**：弱网加载有兜底态（骨架屏/重试），断网操作不丢数据、不出现程序性报错文案
- [ ] **性能达标**：关键路径耗时满足 PRD 非功能需求（如 P95 < 800ms），压测结论已出（预估峰值流量的 2-3 倍）
- [ ] **数据兼容**：涉及数据结构变更时，存量数据迁移/双写方案已验证，脏数据有处理方案

## 四、灰度与开关

- [ ] **功能开关已接入**：新功能被配置开关包裹，开关默认值正确，开/关切换实测生效且用户无异常
- [ ] **灰度方案已配置**：灰度维度（白名单/百分比/渠道）、各阶段比例与放量标准已在发布平台配置完毕
- [ ] **灰度观察指标明确**：每个灰度阶段"看什么指标、达到什么标准才放量"已写死在方案里，不是放量时临时拍
- [ ] **实验冲突检查**：与正在运行的 A/B 实验无流量层冲突，不会污染他人实验

## 五、回滚预案

- [ ] **回滚方式明确并演练过**：开关回滚/版本回滚/配置回滚，选定方式并在预发环境实际演练一次
- [ ] **回滚时长可接受**：从决定回滚到用户侧恢复的时长已知（目标分钟级），审批链路提前打通
- [ ] **回滚的数据后果已评估**：回滚后新产生的数据是否兼容旧逻辑？用户已获得的权益/状态如何处理？
- [ ] **回滚决策人明确**：什么条件触发回滚（量化阈值）、谁有权拍板，写在值班文档里

## 六、监控告警

- [ ] **技术监控就位**：接口成功率、耗时、错误码、崩溃率的监控与告警阈值已配置，告警接收人正确
- [ ] **业务监控就位**：核心业务指标（转化率、下单量等）有分钟/小时级监控，同比环比异常能被发现
- [ ] **资损场景专项监控**（涉及钱时）：金额、发放量、提现等有实时对账或熔断机制
- [ ] **告警演练**：至少手动触发一次告警，确认能真的收到（群/电话），而不是"配置了"

## 七、协作通知

- [ ] **上线通知已发**：上线时间、影响范围、功能说明发到相关群（业务方、客服、市场、数据）
- [ ] **客服口径就绪**：新功能 FAQ 与常见问题话术已交付客服并确认知晓，客诉升级路径明确
- [ ] **值班安排明确**：上线当晚与次日的研发/产品值班人已排定，联系方式在群公告
- [ ] **相关方依赖确认**：依赖的其他团队（风控/支付/审核）知晓上线时间，其服务容量已确认
- [ ] **应用市场/官网物料**（如需要）：版本描述、截图、更新日志已提交

## 常见翻车点提醒

- **埋点是重灾区**：上线后才发现事件漏报/属性为空的比例远高于功能 bug——务必"实际触发并查到数"，不是"代码合了"；
- **开关默认值写反**：灰度开关默认全开导致意外全量，是低级但高发的事故；
- **回滚没演练**：真出事时才发现回滚要走 2 小时审批，预案等于没有；
- **周五晚上上线**：没有特殊理由不要在周五与节前上线，问题会在无人值守时发酵。
