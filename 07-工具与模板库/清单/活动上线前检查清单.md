# 活动上线前检查清单

> 使用说明：活动上线前 48 小时，活动负责人牵头，运营/产品/研发/风控/客服一起逐项过。每项标注负责人与结论；不适用的项标注「N/A + 原因」。涉及现金、实物、抽奖概率的活动，法务与财务两项为硬门槛，未通过不得上线。
>
> 配套模板：[活动方案模板](../模板/活动方案模板.md)、[商业化风控清单](商业化风控清单.md)。

---

## 一、目标与指标

- [ ] **目标已量化并对齐**：核心目标、护栏指标、统计口径已写入方案并经业务负责人确认，避免结束后"成功标准各说各话"
- [ ] **实时看板已就绪**：参与量、成本消耗、核心转化的小时级看板上线前一天可访问，负责人会看
- [ ] **埋点已验证**：活动关键事件（曝光/参与/领奖/分享/提现）在预发环境实际触发并查到数据
- [ ] **数据播报机制明确**：谁、几点、在哪个群播报数据；异常数据（如成本超速）的升级路径明确

## 二、玩法规则与文案

- [ ] **规则穷举边界情况**：断签/并发领取/奖励上限/活动结束瞬间的参与如何处理，规则页均有交代
- [ ] **规则页与实际逻辑一致**：研发按最终版规则实现，文案更新后未出现"页面说 A 代码是 B"
- [ ] **文案合规审查**：无「最」「第一」「100%」等广告法违规用语；不使用「最终解释权归平台所有」（该表述本身涉嫌违规，改为写明具体规则与咨询渠道）
- [ ] **时间信息准确**：活动起止时间（含时区/以何时间为准）、奖励发放时间、余额过期时间在页面显著展示
- [ ] **多端一致**：App/小程序/H5 各端文案、规则、价格一致，分享出去的落地页可正常打开

## 三、利益点与成本审批

- [ ] **预算已走完审批**：总预算、超支 buffer、审批单号留档；口头同意不算数
- [ ] **成本测算含三档**：保守/中性/激进参与量下的成本测算已做，激进档成本公司可承受
- [ ] **预算熔断已配置并演练**：日预算/总预算熔断阈值在系统中配置生效，熔断后的页面降级态已实测
- [ ] **奖励库存核对**：实物/卡券库存数量与页面承诺一致，库存售罄的兜底文案已配置
- [ ] **发放对账机制**：奖励发放流水与财务可对账，T+1 能发现多发/漏发

## 四、风控防刷

- [ ] **薅羊毛路径推演完成**：按 [商业化风控清单](商业化风控清单.md) 逐条推演并留档，风控同学签字确认
- [ ] **参与限制已生效**：单用户/单设备/单手机号/单支付账号的参与上限实测生效
- [ ] **黑产名单接入**：历史作弊名单、风控分接口已接入活动参与与提现链路
- [ ] **奖励发放可拦截**：发现刷量时可对指定用户群冻结/回收奖励，操作路径已演练
- [ ] **提现风控**：提现限频、限额、人机校验就位；提现入口可独立紧急关闭

## 五、技术与压测

- [ ] **压测通过**：按预估峰值 3 倍压测活动页、参与接口、发奖接口，瓶颈已处理并有压测报告
- [ ] **限流与降级预案**：超过承载时的排队/降级静态页已配置，触发阈值明确
- [ ] **活动开关独立**：活动整体、分享入口、提现入口可分别一键关闭，不依赖发版
- [ ] **定时任务核对**：活动自动开始/结束、奖励定时发放的任务时间与时区已核对（大量事故来自时区与 0 点边界）
- [ ] **监控告警**：参与量突刺、发奖失败率、接口错误率的告警已配置且演练收到

## 六、客服口径

- [ ] **FAQ 已交付客服**：规则、常见异常（没收到奖励/提现失败/被判作弊）的标准话术已培训，客服确认知晓上线时间
- [ ] **敏感场景话术统一**：风控冻结用户的话术统一为不透露风控细节的版本，避免教黑产绕过
- [ ] **客诉升级路径**：普通客诉、资损类客诉、舆情级客诉的分级响应人与时限明确
- [ ] **补偿方案预授权**：因我方故障导致的用户损失，客服可用的补偿额度与审批流提前确定

## 七、应急预案

- [ ] **预案文档成文**：预算超支、系统崩溃、羊毛党爆发、舆情发酵四类场景的处置动作、决策人、联系方式写成一页纸
- [ ] **值班表排定**：活动前 3 天高峰期，运营/研发/风控值班人与替补明确，夜间可触达
- [ ] **紧急下线演练**：活动一键下线在预发环境演练过，从决策到用户侧生效的时长已知
- [ ] **对外口径预案**：若需紧急停止活动，公告文案模板提前备好，避免临时措辞失当激化舆情

## 常见翻车点提醒

- **成本失控多发生在前 48 小时**：参与量预估偏差最大的时段，熔断阈值宁紧勿松，人工盯守频次加倍；
- **规则漏洞上线后无法"打补丁"**：活动中途改规则必然引发客诉与舆情，规则边界推演必须在上线前穷尽；
- **风控冻结话术不统一**：客服如实告知"因为您换了设备"等细节，等于给黑产送绕过教程；
- **余额/奖励过期是客诉高峰**：过期前 48 小时必须有 Push/短信提醒，"规则里写了"挡不住舆情；
- **分享链路最后才测**：微信内打开、外部浏览器唤起 App 等分享场景常在上线后才发现挂了，务必真机全路径走一遍。
