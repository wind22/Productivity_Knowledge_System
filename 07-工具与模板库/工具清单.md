# 工具清单

> 使用说明：本清单按用途分类，帮你回答"这类事该用什么工具"。选型总原则：**跟着团队走**（协作工具的价值在于所有人都用）、**先用轻的**（Excel 能解决就别上 BI）、**警惕迁移成本**（数据与文档资产搬家很贵）。每类末尾附一句选型建议。

---

## 一、原型与设计

| 工具 | 一句话说明 | 适用场景与选型建议 |
|---|---|---|
| Figma | 云端协作设计/原型工具，事实上的行业标准 | 团队协作设计、高保真原型、设计交付研发；有设计师的团队首选；国内访问与合规需评估（可考虑替代品 MasterGo/即时设计） |
| 墨刀 | 国产在线原型工具，上手快 | 产品经理独立画中低保真交互原型、快速演示给业务方；不写文档级交互说明时够用 |
| Axure | 老牌原型工具，交互逻辑表达能力最强 | 需要复杂交互逻辑演示（条件跳转、变量）或输出传统交互文档的场景；学习成本高，新团队非必需 |
| ProcessOn / draw.io | 在线流程图/架构图工具 | 画业务流程图、泳道图、脑图；PRD 里的流程图用它或 Mermaid 即可 |

**选型建议**：设计协作用 Figma（或国产替代），PM 自己画原型用墨刀，除非有历史资产否则不必新学 Axure。

## 二、数据处理与分析

| 工具 | 一句话说明 | 适用场景与选型建议 |
|---|---|---|
| SQL | 数据工作的通用语言，从数仓直接取数 | 一切规模化取数与口径定义的基础；产品/运营/分析师都建议掌握到"多表 JOIN + 窗口函数"水平 |
| Excel / Google Sheets | 表格计算与轻量建模 | 万行以内的数据处理、单位经济测算、排期表；优点是人人会看；超过十万行或需复现的分析就该换工具 |
| Python（pandas 等） | 编程式数据分析，可复现、可自动化 | 复杂清洗、统计检验、建模、定时脚本；分析师必备，业务同学可用 AI 辅助写 |
| Tableau | 国际主流 BI，可视化能力与交互探索最强 | 分析师做探索式分析与高质量可视化；授权费较高，适合数据团队成熟的公司 |
| Power BI | 微软系 BI，与 Office/Azure 生态集成好 | 已深度使用微软全家桶的企业首选；性价比高于 Tableau |
| Quick BI | 阿里云 BI，与国内云数仓（MaxCompute 等）打通顺畅 | 数据在阿里云上的国内公司做经营看板；同类可比 FineBI、观远 |

**选型建议**：先保证 SQL + Excel 这条“地板组合”人人过关；BI 跟着公司云与预算走，别为工具而工具——看板没人看的问题永远不出在 BI 选型上。

## 三、协作与文档

| 工具 | 一句话说明 | 适用场景与选型建议 |
|---|---|---|
| 飞书 | IM + 文档 + 多维表格 + 审批一体的协作套件 | 国内互联网团队首选之一：PRD、周报、需求池（多维表格）、审批流一站式；本知识库多数模板可直接搬进飞书文档/多维表格 |
| Notion | 块编辑器 + 数据库的知识管理工具 | 个人知识库、小团队 Wiki、数据库化管理需求池/实验记录；国内访问稳定性与协作规模是主要顾虑 |
| 语雀 | 蚂蚁出品的结构化知识库工具 | 沉淀团队知识库、对外文档；文档树结构清晰，适合"库"型内容；IM 协同弱于飞书 |
| Confluence + Jira | Atlassian 套件：Wiki + 项目管理 | 研发流程重、有跨国协作或已有 Atlassian 资产的团队；新团队一般不再首选 |

**选型建议**：团队用什么你用什么；有选择权时，国内团队「飞书」大而全最省心，知识库沉淀可加语雀，个人笔记随意。

## 四、埋点与用户行为分析平台

| 工具 | 一句话说明 | 适用场景与选型建议 |
|---|---|---|
| GA4（Google Analytics 4） | Google 免费流量与行为分析，事件模型 | 出海产品 / 独立站标配，与 Google Ads 归因打通；国内 App 基本不可用 |
| 神策数据 | 国产用户行为分析，支持私有化部署 | 对数据安全要求高（金融/政企/大厂）、需要私有化与全埋点+代码埋点结合的团队；付费，功能全 |
| GrowingIO | 国产增长分析平台，以无埋点起家（已并入创略等生态） | 想低成本快速看转化漏斗、运营团队自助分析的中型团队；无埋点省研发但口径精度弱于代码埋点 |
| 友盟+ | 阿里系移动统计，免费额度大 | 初创 App 的第一套统计（新增/活跃/崩溃）；深度行为分析与开放性有限，长大后通常迁移 |
| 火山引擎 DataFinder / Amplitude | 字节系 / 国际系行为分析 | 预算充足、重增长实验的团队备选：国内看 DataFinder，海外看 Amplitude |

**选型建议**：早期免费起步（国内友盟+，海外 GA4）；进入精细化运营阶段再上神策/GrowingIO 这类付费平台，并且**先建埋点规范与[指标字典](表格/指标字典模板.md)再换工具**——平台再贵也救不了口径混乱。

## 五、效率补充（可选）

| 工具 | 一句话说明 | 适用场景 |
|---|---|---|
| Xmind / 幕布 | 思维导图 | 需求脑暴、结构化拆解、汇报大纲 |
| 问卷星 / 腾讯问卷 | 问卷调研 | 用户调研、NPS 收集；注意样本偏差 |
| TAPD / PingCode | 国产研发项目管理 | 需求-迭代-缺陷全流程管理，腾讯系团队多用 TAPD |

## 选型决策三问（拿不准时自查）

1. **这个问题现在的工具真的解决不了吗？**——多数"想换工具"其实是流程或规范问题；
2. **团队里最不擅长工具的人能用起来吗？**——协作工具的下限决定其价值；
3. **两年后要迁移，数据和文档能导出来吗？**——警惕导出能力差的封闭工具。

## 按团队阶段的推荐组合

| 阶段 | 推荐组合 | 说明 |
|---|---|---|
| 0-10 人初创 | 飞书（文档+表格+IM）+ 友盟+/GA4 + Excel | 全部免费或近免费；把钱和时间花在业务上 |
| 10-50 人成长期 | 上一档 + SQL 数仓自助取数 + 墨刀/Figma + TAPD | 关键动作：建立指标字典与埋点规范，先于买 BI |
| 50 人以上成熟期 | + 神策/GrowingIO + Tableau/Quick BI + 实验平台 | 工具预算换人效；专人负责数据平台与口径治理 |

## 不同角色的最小工具栈

| 角色 | 必备 | 加分 |
|---|---|---|
| 产品经理 | 飞书/文档、墨刀或 Figma、SQL 基础、Xmind | Python 基础、埋点平台自助分析 |
| 运营 | 飞书多维表格、Excel、埋点平台自助分析 | SQL、问卷工具、简单图片处理 |
| 数据分析师 | SQL（熟练）、Python、一款 BI | 统计检验功底 > 工具数量 |
| 商业分析 | Excel 建模（熟练）、PPT/文档叙事、SQL | 行业数据库（wind/企查查等）使用 |

> 提醒：工具清单会过时（价格、功能、归属变化快），本文观点以"选型逻辑"为主、具体产品为辅；重大采购前请以官网最新信息与试用结论为准。
