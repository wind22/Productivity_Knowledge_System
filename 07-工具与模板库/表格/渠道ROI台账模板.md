# 渠道 ROI 台账模板

> 使用说明：投放台账建议按「渠道 × 周」粒度记录（预算大的渠道可按日）。台账的价值在于坚持记录形成时间序列——单周数据说明不了问题，趋势才说明问题。
>
> 维护规则：
> 1. 花费以渠道后台实际扣费为准，激活/次留以我方归因平台为准，两边对不上时在备注注明差异率；
> 2. LTV 预估口径全表统一（本模板用 90 日 LTV 预估），换口径等于重开一张表；
> 3. 每周投放例会以本台账为唯一数据源，杜绝"各说各话"。

---

## 一、字段说明与计算公式

| 字段 | 说明 / 公式 |
|---|---|
| 渠道 | 渠道名+子计划（如「渠道X-信息流-视频素材A」），粒度视预算而定 |
| 时间 | 统计周期（周或日），按激活时间归属，不按点击时间 |
| 花费（元） | 渠道后台实际消耗，含服务费 |
| 曝光 | 广告展示次数 |
| 点击 | 广告点击次数；CTR = 点击 / 曝光 |
| 激活 | 归因到该渠道的新增激活设备数（去重、剔除作弊） |
| 次留 | 该批激活用户的次日留存率 = 次日活跃数 / 激活数 |
| CAC（元） | 单个激活成本 = 花费 / 激活 <!-- 若考核注册或首购，分母换成对应事件并注明 --> |
| LTV预估（元） | 该渠道用户 90 日预估生命周期价值 = 90 日预估留存曲线下面积 × 日均 ARPU（由数据组按渠道历史拟合，每月更新一次系数） |
| ROI | = LTV预估 / CAC。>1 表示 90 日口径下回本；结合毛利率看真实盈亏：毛利口径 ROI = LTV × 毛利率 / CAC |
| 备注 | 素材更换、竞价调整、异常说明、与渠道后台的数据差异 |

**衍生指标速查**：CTR = 点击/曝光；CVR（点击到激活）= 激活/点击；CPM = 花费/曝光×1000。转化链路异常时先看 CTR 和 CVR 哪一环变了。

## 四、使用与决策建议

- **放量/缩量判断**：不看单周 ROI 拍板。连续 2-3 周 ROI < 1（毛利口径）且优化无改善 → 缩量或暂停；ROI 健康且次留稳定 → 阶梯放量（每次不超过 50%），放量后紧盯次留是否被稀释；
- **警惕平均数陷阱**：混合 CAC 变好可能只是低价低质渠道占比升高，务必下钻到渠道 × 素材层看；
- **LTV 预估要回测**：每季度用真实 90 日 LTV 回测预估误差，误差 >20% 时重新拟合系数，并在备注中标记受影响周期；
- **数据打架处理**：渠道后台数据普遍高于我方归因（归因窗口、口径差异），以我方归因做决策、渠道后台做对账，差异率 >15% 时排查作弊。

## 五、渠道质量异常排查清单

当某渠道 CAC 骤降或激活量突增（"好得不正常"）时，按序排查：

- [ ] **次留/7留是否同步崩塌**：量涨留存崩是买量作弊或人群外溢的典型信号
- [ ] **激活时间分布**：真实用户激活分布在全天，机刷常集中在深夜或整点
- [ ] **设备特征**：新增中低价机型/老系统版本/同 IP 段占比是否突增
- [ ] **行为深度**：激活后完成注册、核心行为的比例是否显著低于该渠道历史水平
- [ ] **地域分布**：与投放定向地域是否吻合
- [ ] 确认作弊后：截图留证 → 与渠道对质扣量/索赔 → 该渠道进入观察名单，降低预算权重

## 六、口径备忘（首次建表时确定，写死在此处）

| 口径项 | 本表约定 | 备注 |
|---|---|---|
| 归因模型 | *last-touch，点击后 24h / 展示后 6h* | 换归因模型历史数据不可比 |
| 激活定义 | *首次启动并初始化 SDK 成功* | 若考核注册/首购须另开列 |
| LTV 口径 | *90 日预估、收入口径*（毛利口径另列） | 系数每月由数据组更新 |
| 作弊剔除 | *风控标记设备不计入激活与分母* | 与渠道结算口径可能不同，对账注明 |
| 汇率/币种 | *人民币；海外渠道按扣费当日汇率折算* | |
