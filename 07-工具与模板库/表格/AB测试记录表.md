# A/B 测试记录表

> 使用说明：所有实验（含失败与不显著的）都必须登记本表——不显著的实验同样是知识资产，能防止团队半年后重做同一个实验。
>
> 使用规则：
> 1. **假设先行**：实验启动前填完「假设、指标、样本量、起止时间」并锁定，禁止实验中途改指标或提前偷看后停止（peeking 会使 p 值失真）；
> 2. 假设写成可证伪格式：「若对【人群】做【改动】，则【核心指标】将【方向+幅度】，因为【机制】」；
> 3. 护栏指标必填：核心指标涨了但护栏跌了不算赢；
> 4. 决策只有四种：全量 / 放弃 / 迭代后重测 / 延长实验（写明延长理由与新截止日）。

---

## 一、字段说明

| 字段 | 填写要求 |
|---|---|
| 实验ID / 实验名 | 唯一编号 + 一眼可懂的名字 |
| 假设 | 可证伪格式（见上）；机制说不清的实验大概率白做 |
| 核心指标 | 只允许 1 个，用于判断成败；口径引用 [指标字典](指标字典模板.md) |
| 护栏指标 | 1-3 个不允许恶化的指标（如收入实验护栏是留存/客诉） |
| 样本量计算 | 记录：基线值、MDE（最小可检测提升）、显著性水平 α（常用 0.05）、功效 1-β（常用 0.8）→ 所需每组样本量与预计运行天数 |
| 分组与流量 | 分组数、各组流量比例、分流单位（用户/设备/会话）、互斥层 |
| 起止时间 | 计划与实际都记；不满一个完整周期（通常 ≥7 天）的结果不采信 |
| 结果 | 核心指标提升幅度（相对%）、95% 置信区间、p 值；护栏指标结论 |
| 决策 | 全量 / 放弃 / 迭代重测 / 延长，附一句决策理由 |
| 经验沉淀 | 无论成败，写一条"我们学到了什么"（关于用户的认知，而非关于实验的流水账） |

## 三、样本量与运行时长速查

<!-- 实验启动前用在线计算器（如 Evan Miller 的 Sample Size Calculator）或让数据同学用 power analysis 算，把参数记入表格。此处给直觉参考 -->

**所需样本量的直觉规律**（α=0.05，功效 0.8，转化率类指标）：

| 基线转化率 | 想检测的相对提升 | 每组约需样本 |
|---:|---:|---:|
| 20% | 10%（即 +2pp） | 约 6,300 |
| 20% | 5%（即 +1pp） | 约 25,000 |
| 5% | 10%（即 +0.5pp） | 约 30,000 |
| 5% | 5%（即 +0.25pp） | 约 120,000 |

要点：

- MDE 减半，样本量约变为 4 倍——"想看出微小差异"的代价是平方级的；
- 预计运行天数 = 每组所需样本 ÷ 每组日均进入实验人数，向上取整到完整周；
- 算出来要跑 6 周以上？说明流量不够或 MDE 定得太苛刻，考虑放大改动幅度、聚焦高流量场景，或接受更大的 MDE。

## 四、决策规则（实验结束时对号入座）

| 核心指标 | 护栏指标 | 决策 |
|---|---|---|
| 显著为正 | 无恶化 | 全量 |
| 显著为正 | 显著恶化 | 权衡取舍，默认不全量；量化两边影响后升级决策 |
| 不显著 | 无恶化 | 看点估计方向与业务成本：改动成本低可全量（简化产品），否则放弃 |
| 显著为负 | — | 放弃，并把"为什么假设错了"写入经验沉淀 |
| 样本未达计划量 | — | 不下结论；延长或放弃，禁止拿中途数据当结论 |

## 五、常见坑自查（结果分析前过一遍）

- [ ] **AA 校验 / SRM 检查**：实验组与对照组样本比例是否符合设定（偏差用卡方检验，SRM 异常则结果作废）
- [ ] **完整周期**：是否覆盖 ≥1 个完整周（工作日/周末行为不同）；大促等异常期数据是否剔除或注明
- [ ] **新奇效应**：改动显眼的实验，前 3 天效应是否明显衰减？建议看分日趋势而非只看累计值
- [ ] **多重比较**：同时看了很多细分维度？细分结论只作假设，需单独实验验证，不直接下结论
- [ ] **辛普森悖论**：大盘显著但主要细分方向相反时，先查流量结构再下结论
- [ ] **溢出效应**：社交/交易类产品实验组行为是否会影响对照组（必要时用地理/时间片分流）

## 六、季度回顾

每季度盘点一次本表：胜率（全量实验占比）、平均提升幅度、被推翻的直觉假设 Top3。
健康的实验文化胜率通常只有 20-30%——如果胜率长期高于 50%，多半是只登记了成功的实验，或改动过于保守。
把「经验沉淀」列单独汇总成《我们对用户的认知清单》，这是实验体系最值钱的产出。
