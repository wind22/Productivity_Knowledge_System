# 指标字典模板

> 使用说明：指标字典是团队数据口径的"唯一事实来源"。所有报表、分析报告、实验结论引用指标时，都应以本字典为准；口径变更必须走变更记录并通知下游。
>
> 维护规则：
> 1. 每个指标必须有唯一负责人（业务口径归业务方，计算口径归数据方，表中分开写也可）；
> 2. 「计算口径」精确到可直接复现——分子分母、去重键、过滤条件、时间边界缺一不可，配 SQL 示例最佳；
> 3. 新增指标先查重，避免"同名不同义、同义不同名"；
> 4. 口径核对时配合使用 [数据口径核对清单](../清单/数据口径核对清单.md)。

---

## 一、字段说明

| 字段 | 填写要求 |
|---|---|
| 指标名 | 中文名 + 英文缩写，全团队统一，禁止一指标多名 |
| 业务定义 | 一句人话：这个指标衡量什么业务问题 |
| 计算口径 | 分子/分母、去重键（设备/用户/账号）、过滤条件、时间边界（自然日？时区？）；附可运行的 SQL 示例 |
| 数据来源 | 具体到库表名与更新时效（T+1 / 实时） |
| 负责人 | 口径解释与变更的第一责任人 |
| 更新频率 | 指标产出频率（日更/周更/实时） |

## 三、SQL 示例附录

**#1 DAU**

```sql
SELECT dt,
       COUNT(DISTINCT COALESCE(user_id, device_id)) AS dau
FROM dws.app_active_di
WHERE dt = '${bizdate}'
  AND is_internal = 0        -- 剔除内部账号
  AND risk_flag = 0          -- 剔除风控标记
GROUP BY dt;
```

**#2 次日留存率**

```sql
SELECT a.dt,
       COUNT(DISTINCT r.device_id) / COUNT(DISTINCT a.device_id) AS d1_retention
FROM dwd.user_new_di a                     -- 当日新增（分母）
LEFT JOIN dws.app_active_di r              -- 次日活跃（分子）
       ON a.device_id = r.device_id
      AND r.dt = DATE_ADD(a.dt, 1)
WHERE a.dt = '${bizdate}'
  AND a.cheat_flag = 0
GROUP BY a.dt;
```

**#3 GMV**

```sql
SELECT dt, SUM(payable_amount) AS gmv     -- 应付金额，不含运费
FROM dwd.trade_order_di
WHERE dt = '${bizdate}'
  AND order_status = 'PAID'               -- 支付成功口径，退款不回溯
GROUP BY dt;
```

**#4 支付转化率**

```sql
SELECT dt,
       COUNT(DISTINCT IF(step = 'pay_success', user_id, NULL))
     / COUNT(DISTINCT IF(step = 'item_view',  user_id, NULL)) AS pay_cvr
FROM dws.funnel_pay_di
WHERE dt = '${bizdate}'
GROUP BY dt;
```

**#5 ARPU（月）**

```sql
SELECT month,
       SUM(pay_amount) / MAX(mau) AS arpu_monthly   -- 分母为 MAU，注意与日 ARPU 区分
FROM ads.revenue_summary_df
WHERE month = '${bizmonth}'
GROUP BY month;
```
