# 第2章 产品

> 📍 所属：第2章 产品 · 层级：导航 · 本章帮助你建立从"产品本质"到"日常实操"的完整产品能力体系

## 本章结构：五层模型

产品知识不是零散技巧的堆砌，而是一座金字塔。本章按照「原理 → 概念 → 方法 → 技巧 → 误区」五层展开，越靠近底层越稳定，越靠近顶层越具体：

```
        ┌─────────────┐
        │  2.5 常见误区  │  ← 反面教材：别人踩过的坑
        ├─────────────┤
        │  2.4 关键技巧  │  ← 实战经验：让方法落地更顺滑
        ├─────────────┤
        │  2.3 方法论   │  ← 可执行流程：KANO、PRD、竞品分析…
        ├─────────────┤
        │  2.2 核心概念  │  ← 通用语言：MVP、PMF、网络效应…
        ├─────────────┤
        │  2.1 第一性原理 │  ← 不变的本质：产品/需求/体验是什么
        └─────────────┘
```

| 层级 | 文档 | 解决什么问题 | 适合什么时候读 |

| 原理 | [2.1 第一性原理](./2.1-第一性原理.md) | 产品、需求、体验、产品经理工作的本质是什么 | 入行第一天；每次感到迷茫、方向摇摆时回来重读 |

| 方法 | [2.3 方法论](./2.3-方法论.md) | 从需求收集到 PRD 落地的全流程可执行方法 | 接到具体任务（做调研、写 PRD、做规划）时按图索骥 |

## 推荐阅读路径

- **新人（0-1 年）**：2.1 → 2.2 → 2.5（先立骨架、先知道坑在哪），再在实际任务中查阅 2.3。
- **进阶（1-3 年）**：重点精读 2.3 与 2.4，用 2.5 的自检问题做每次立项的例行检查。
- **转岗者（运营/数据转产品）**：先读 2.1 校准视角，再读 2.2 补齐术语，结合 [第3章 运营](../03-运营/) 与 [第4章 数据分析](../04-数据分析/) 的既有知识迁移。
