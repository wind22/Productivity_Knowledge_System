# 5.5 商业分析常见误区

> 📍 所属：第5章 商业分析 · 层级：误区 · 本文拆解商业分析中最高频的四个坑：只看收入增长、TAM 拍脑袋、伪护城河、框架堆砌——每个误区按「表现→为什么会踩→如何避免→自检问题」展开，可当作交付前的自检清单。

## 误区一：只看收入增长，不看单位经济

### 表现

- 汇报和 BP 里通篇 GMV/收入/用户数的增长曲线，对"每单赚不赚钱"避而不谈；
- 用"规模起来了自然会盈利"回应所有质疑，但说不出哪项成本会随规模下降、降到多少；
- 增长目标倒逼补贴加码，补贴一停数据崩盘，于是继续补——把补贴出来的留存当真实留存；
- 评估竞对时惊呼"它增速 200%"，却不看它每一块收入亏多少钱。

### 为什么会踩

- **激励结构**：增长指标（GMV、DAU）是团队 KPI，单位经济是财务的事——组织分工天然制造盲区；
- **叙事惯性**：资本充裕期"先规模后盈利"的成功案例（京东、美团）被过度泛化，忽略了它们的单位经济从第一天起就有清晰的改善斜率；
- **数字的迷惑性**：增长是每天看得见的正反馈，单位经济恶化是慢变量，人天然重视快变量。

### 如何避免

1. 任何增长汇报强制附一页单位经济：单均收入/变动成本/毛利 + 各项的规模化方向（模板见 [5.4 关键技巧](5.4-关键技巧.md)）；
2. 区分两种亏损：**结构性亏损**（成本项不随规模下降，如上门 O2O 的在途时间）必须止损；**投资性亏损**（成本项有明确下降路径，如订单密度摊薄配送成本）可以继续；
3. 把"剔除补贴后的留存/复购"设为正式口径，与含补贴口径并列汇报；
4. 记住社区团购的教训：千亿补贴烧出来的规模，在监管与资本退潮后迅速坍缩，因为生鲜的单位经济（高损耗+低客单+高履约）从未被规模改善。

### 自检问题

- [ ] 我能一口报出这个业务的单均毛利吗？是正是负？
- [ ] 亏损的每一项成本，规模翻 10 倍后分别变成多少？依据是什么？
- [ ] 停掉全部补贴，下个月的留存/复购会掉多少？
- [ ] LTV/CAC 用的生命周期假设，有几个月的真实留存数据支撑？

## 误区二：TAM 自上而下拍脑袋

### 表现

- "中国餐饮市场 5 万亿，我们做到 1% 就是 500 亿"——经典句式，1% 从何而来无人追问；
- 引用行业报告的最大口径数字（把"泛人群"当"目标客群"），层层百分比全靠感觉；
- 只做自上而下，从不做自下而上交叉验证；
- 混用市场层级：用 TAM 讲故事，却直接把 TAM 当作自己的经营天花板做预测。

### 为什么会踩

- **动机污染**：BP 要融资、立项要过会，大数字有说服力——测算在动笔前就被结论绑架；
- **自上而下太容易**：搜一份报告乘几个百分比只要十分钟；自下而上要定义客群、验证客单和频次，要几天；
- **对"可服务"缺乏敬畏**：理论需求 ≠ 你的产品形态、价格带、渠道能触达的需求（TAM/SAM/SOM 的层级定义见 [5.2 核心概念](5.2-核心概念.md)）。

### 如何避免

1. **强制双路径**：自上而下和自下而上各算一遍，差异超过 3 倍就逐项排查假设——这是最有效的单条纪律；
2. 每个百分比写清来源与依据，写不出来的标红为"待验证假设"而非事实；
3. 用"当前实际渗透"校准：找一个已被打透的对标城市/客群，用真实数据反推全国，比任何报告都可靠；
4. 警惕报告口径：行业报告的"市场规模"常含 B 端、线下、存量翻新等与你无关的部分，引用前先读统计口径的脚注。

### 自检问题

- [ ] 我的 SOM 是自下而上算出来的，还是 TAM 乘了一个"看起来谦虚"的百分比？
- [ ] 两种路径的测算差异在 3 倍以内吗？
- [ ] 模型里最大的那个百分比假设，敢不敢拿去做用户调研验证？
- [ ] 如果把"目标客群"从泛人群收窄到"付得起且付过钱的人群"，数字缩水多少？

## 误区三：把暂时领先误认为护城河

### 表现

- 把先发优势、市场份额、增长速度、融资额、团队执行力写进"我们的壁垒"；
- 竞对分析里写"对手需要 2 年才能追上我们的功能"，把时间差当作结构性优势；
- 对新进入者反应迟钝："这个市场我们已经赢了"；
- 战略讨论中"技术领先"永远是壁垒清单第一条，但说不清专利/数据/工程哪一环不可复制。

### 为什么会踩

- **幸存者叙事**：赢的时候所有动作都被追认为"壁垒"，混淆了相关与因果；
- **概念不清**：不了解护城河的严格定义——**竞争者即使看清了你的打法、拿到同等资源，依然无法夺走你的客户**（五种真护城河见 [5.1 第一性原理](5.1-第一性原理.md)）；先发、执行、融资只是"挖河的时间窗口"，不是河本身；
- **静态视角**：护城河会变窄——渠道变迁（货架电商 vs 短视频电商）、技术代际（功能机 vs 智能机）都会把旧河填平。

### 如何避免

1. 用"十倍资金测试"逐条过壁垒清单：**给竞争者 10 倍资金和完整信息，这条优势还在吗？** 在的才是护城河；
2. 用转换成本的视角检验：用户/客户离开你要付出什么具体代价（数据、关系链、学习成本、合约）？说不出代价=没有转换成本；
3. 每季度问一次动态问题：我们的河在变宽还是变窄？证据是什么？（例：滴滴的规模领先被聚合平台稀释，见 5.1 的案例）；
4. 区分"运营领先"与"结构壁垒"的正确用法：运营领先决定你**现在**赢，结构壁垒决定你**一直**赢——前者的价值在于争取时间把它转化为后者。共享单车是反面教材：颜色大战中没有任何一家把规模转化为转换成本或网络效应，终局是全行业为供应商打工。

### 自检问题

- [ ] 我列的每条"壁垒"，属于五种护城河（品牌/网络效应/转换成本/成本优势/规模+牌照）中的哪一种？归不进去的删掉；
- [ ] 给对手 10 倍资金，哪几条会失效？
- [ ] 我们的用户离开时，具体损失什么？我能写出来吗？
- [ ] 过去一年有没有出现让这条河变窄的渠道/技术/监管变化？

## 误区四：战略框架堆砌，没有 So What

### 表现

- 40 页 PPT：SWOT、五力、PEST、BCG 一个不少，最后一页结论是"建议持续关注、加强投入"；
- 框架的格子填得满满当当，但填完之后的推导为零——SWOT 只列四格不做交叉策略，五力打完分不回答"所以利润会流向哪"；
- 先选框架再找信息硬塞，而不是先有问题再选工具；
- 汇报后老板问"所以我们该做什么？"，答不上来。

### 为什么会踩

- **框架给人安全感**：填格子有明确的完成感，而"给出判断"要承担错误的风险——框架堆砌本质是**用工作量回避判断责任**；
- **误解框架的用途**：框架是组织信息的脚手架，不是分析本身；分析=从信息推出**非显然的判断**和**可执行的动作**；
- **缺少收敛训练**：学校和咨询入门教的是发散（MECE、全面），但商业决策要的是收敛（关键驱动因素 + 取舍）。

### 如何避免

1. **动笔前写下那个问题**：本次分析要回答的决策问题是什么（进不进？投多少？打不打价格战？）——所有内容为回答它服务，无关的框架直接不做；
2. **每个框架强制产出一句 So What**：五力分析的最后一行必须是"因此行业利润将流向____，我们的位置是____"；SWOT 必须推导出 SO/WO/ST/WT 四类策略（见 [5.3 方法论](5.3-方法论.md)）；
3. **结论先行（金字塔原理）**：第一页就是"结论 + 三个依据 + 建议动作"，框架全部进附录；
4. **用"电梯测试"自检**：30 秒说不清结论和依据，说明分析还没做完——不是内容不够，是判断没长出来。

### 自检问题

- [ ] 这份分析回答的决策问题，我能用一句话说出来吗？
- [ ] 删掉所有框架图表后，剩下的判断句有几句？（少于 3 句=还在堆砌）
- [ ] 我的建议具体到"谁、在什么时间、做什么、投入多少"了吗？
- [ ] 如果我的结论是错的，最可能错在哪个假设？（答不出=没有真判断，见 [5.4](5.4-关键技巧.md) 敏感性分析）

## 四个误区的共同根源

| 误区 | 表面问题 | 共同根源 |
|------|---------|---------|
| 只看收入增长 | 指标选择性失明 | 用容易获得的信号替代关键信号 |
| TAM 拍脑袋 | 测算不严谨 | 结论先行，分析为叙事服务 |
| 伪护城河 | 概念误用 | 把相关当因果，把现状外推为终局 |
| 框架堆砌 | 没有判断 | 用工作量回避判断责任 |

一句话总结：**商业分析的交付物是"判断 + 依据 + 动作"，其余一切（框架、数据、图表）都是原材料。** 交付前用四组自检问题过一遍，比多加十页附录有用得多。

## 附：交付前 60 秒总自检清单

任何商业分析文档（竞对报告、立项书、BP、策略汇报）发出前，快速过这七问：

1. 结论能否用一句话说清？放在第一页了吗？
2. 收入/增长数字旁边，有对应的单位经济数字吗？
3. 市场规模做过自下而上验证吗？两条路径差异在 3 倍以内吗？
4. "壁垒"清单里的每一条，都能归入五种护城河吗？
5. 每个框架图后面，都跟着一句 So What 吗？
6. 建议动作具体到了"谁、何时、做什么、投入多少"吗？
7. 我知道自己的结论对哪个假设最敏感吗？该假设验证了吗？

七问中有两问以上答"否"，先补分析再发——被挑战一次的信誉成本，远高于晚发半天。
