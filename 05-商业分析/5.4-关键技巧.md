# 5.4 商业分析关键技巧

> 📍 所属：第5章 商业分析 · 层级：技巧 · 本文是"明天就要交付"的实战手册：快速算账判断业务成不成立、三步拆财报、竞对情报的一手来源、反共识找机会、BP 标准结构、敏感性分析——每个技巧都给出可直接套用的步骤和示例。

## 一、用单位经济模型快速判断"能不能算得过账"

### 概念与场景

老板扔来一个新业务想法，或者你在评估一个竞对/创业项目，第一件事不是做 40 页 PPT，而是**用 30 分钟在一张纸上算清一单的账**。单位经济为负且无改善路径的业务，后面的一切分析都不必做（概念定义见 [5.2 核心概念](5.2-核心概念.md)）。

### 操作步骤

1. 定义最小业务单元（一单/一个订阅用户/一家门店）；
2. 列出单均收入的每一项、单均**变动成本**的每一项（固定成本另算盈亏平衡）；
3. 算单均贡献毛利 → LTV → LTV/CAC；
4. 关键一步：**标出每项在规模化后的变动方向**（↑/↓/→），这决定"现在亏"是战略还是绝症。

### 测算示例：自营咖啡外送品牌（每一杯的账）

| 项目 | 金额（元） | 规模化方向 | 备注 |
|------|----------|-----------|------|
| 单均收入（客单/杯数折算） | 16.0 | → | 含配送费分摊 |
| 原料+包材 | −5.0 | ↓ | 采购规模议价，可降至 4.5 |
| 配送成本分摊 | −4.5 | ↓ | 订单密度提升后可至 3.5 |
| 平台佣金/支付 | −1.3 | → | 费率刚性 |
| 促销补贴分摊 | −3.0 | ↓ | 成熟期目标 1.0 |
| **单均贡献毛利** | **+2.2** | **↑** | 成熟期模型约 +5.7 |

```
LTV  = 单均毛利 2.2 × 月均 3 单 × 生命周期 14 个月 ≈ 92 元
CAC  = 市场费用 60 万/月 ÷ 新客 1.5 万/月 = 40 元
LTV/CAC = 2.3 → 当前略低于健康线 3
但成熟期模型：LTV ≈ 5.7 × 3 × 14 ≈ 239，LTV/CAC ≈ 6 → 值得投入
```

**结论模板**："当前单均毛利 +2.2 元、LTV/CAC=2.3，暂不健康；但补贴与配送两项有明确的下降路径，成熟模型 LTV/CAC≈6，账算得过来，建议在 3 个城市验证密度假设后再扩张。"——注意结论必须落到**动作**。可直接套用 [单位经济模型测算表](../07-工具与模板库/表格/单位经济模型测算表.md)。

### 快速判断的三个红旗

- 单均毛利为负，且**每一项成本的规模化方向都是 → 或 ↑**（如上门按摩的技师在途时间）；
- LTV 依赖"生命周期 36 个月"这类没有留存数据支撑的假设；
- CAC 用的是"自然流量为主时期"的数据，规模化必然靠付费投放。

## 二、拆财报三步法：30 分钟看懂一家公司的年报

财报几百页，分析师真正读的是三层。按顺序拆，每步回答一个问题。

### 第一步：收入结构与增速 —— 钱从哪来？还在长吗？

- 找**分部收入表**（Segment Revenue），看各业务线占比与各自增速；
- 关键动作：把"总收入增速"拆成"哪块在拉、哪块在拖"。总增速 20% 可能是老业务 5% + 新业务 200% 的混合，两者含义完全不同；
- 追问：增长来自量（用户/订单）还是价（ARPU/take rate）？靠价驱动的增长天花板近。

### 第二步：毛利与费用率 —— 赚钱的质量如何？

- 毛利率趋势 > 毛利率绝对值：毛利率连续下滑=定价权在流失或结构在变差；
- 三费率（销售/管理/研发 ÷ 收入）看**经营杠杆**：收入涨 30% 而费用只涨 10%，利润会加速释放——这是平台型公司利润拐点的典型信号（参考美团 2019、拼多多 2021 的扭亏路径：收入过了固定成本临界点，销售费用率从 100%+ 一路降到 40% 以下）；
- 销售费用率还是**增长质量测谎仪**：收入增速 30% 但销售费用增速 60%，说明增长是买来的。

### 第三步：现金流质量 —— 利润是真的吗？

- **经营现金流 vs 净利润**：长期"利润高、现金流低"，查应收账款和收入确认政策（乐视警报）；"利润低、现金流高"，往往是好模式（京东：账期占用上游资金；Costco：会员费预收）；
- **自由现金流** = 经营现金流 − 资本开支：烧钱扩张期为负可接受，但要看单位资本开支换来的收入增量是否递减；
- **递延收入变化**：订阅/SaaS 公司的领先指标，递延收入增速 < 确认收入增速 = 未来收入将放缓的预警。

> 三步口诀：**一看钱从哪来，二看赚钱质量，三看真金白银**。财报之外，务必配套读"管理层讨论与分析（MD&A）"和电话会问答——数字说明发生了什么，管理层的措辞变化暗示将要发生什么。

## 三、竞对情报来源清单：合法且高价值的一手渠道

| 来源 | 能挖到什么 | 使用技巧 |
|------|-----------|---------|
| 财报/招股书 | 收入结构、成本、风险因素、募资投向 | 招股书"风险因素"和"竞争"章节是公司自己写的弱点清单；港/美股招股书行业部分引用的第三方数据可白嫖 |
| 业绩电话会（Earnings Call） | 战略优先级、指引、管理层态度 | 重点读 Q&A 环节的回避性回答；对比连续四个季度的措辞变化 |
| 招聘 JD | 未公开的新业务、技术路线、组织扩张方向 | 竞对突然大量招"海外支付风控"=出海支付业务在启动；JD 里的系统名/合作方名是情报金矿 |
| 专利与软著 | 技术储备与产品方向 | 专利公开有 18 个月滞后，适合看中期方向而非追热点 |
| 渠道调研 | 真实销售政策、返点、库存水位 | 以客户身份询价（合规）、访谈离职渠道商；B2B 行业最有效的一手信息 |
| 应用商店数据 | 版本节奏、下载量级、评论抱怨 | 第三方平台（七麦、点点、Sensor Tower）看下载与收入估算；差评关键词聚类=对手的软肋清单 |
| 其他 | 管理层访谈/播客、行业展会、裁判文书网（诉讼）、企查查（股权与对外投资） | 对外投资清单是集团战略意图的显影剂 |

**纪律红线**：只用公开与合法渠道，不碰内部资料、不诱导在职员工泄密——除了法律风险，被污染的情报本身不可信。

## 四、用"反共识"寻找机会

### 概念

彼得·蒂尔的问题："**有什么重要的事实，很少有人同意你？**" 机会的大小 = 你正确的程度 × 市场不同意你的程度。共识里没有超额收益：所有人都知道的机会，其回报已被竞争抹平。

### 操作方法

1. **列出行业共识**：把从业者张口就来的判断写下来（"下沉市场没有消费力""年轻人不看长内容""电商格局已定"）；
2. **逐条找反例与数据裂缝**：共识通常基于过时的约束条件——找到"约束条件已变化但认知没跟上"的点；
3. **验证是"反共识且正确"还是"反共识且错误"**：用最小成本实验验证（这是数据分析登场的地方，见 [04-数据分析](../04-数据分析/)）。

### 互联网示例

- 拼多多（2015）：共识是"电商格局已定、消费在升级"；反共识是**微信支付刚打通的数亿下沉用户没有被服务，且他们要的是绝对低价而非升级**；
- B 站破圈前的共识是"二次元社区做不大"；反共识是"Z 世代的学习与泛知识需求缺一个视频承载地"；
- 注意反例：社区团购大战中"共识错误"的部分（生鲜电商难）其实是对的，反共识者集体阵亡——**反共识不是逆共识，多数共识是对的**，这也是该技巧最危险的地方。

## 五、BP（商业计划书）结构：八段论

无论是融资 BP 还是内部立项书，逻辑链一致：**问题→方案→市场→竞争→商业模式→团队→财务预测→融资计划**。每段的核心任务与常见死法：

| 段落 | 必须回答 | 常见死法 |
|------|---------|---------|
| 1. 问题 | 谁的什么痛点？现在怎么凑合解决？痛到愿意付钱吗？ | 伪需求、低频轻痛 |
| 2. 方案 | 你的解法为什么好 10 倍（而非 10%）？ | 只是"更好一点" |
| 3. 市场 | TAM/SAM/SOM，自下而上算（见 [5.2](5.2-核心概念.md)） | 拍脑袋 TAM |
| 4. 竞争 | 竞争地图 + 你的护城河计划 | 写"我们没有竞争对手" |
| 5. 商业模式 | 谁付钱、怎么收、单位经济 | 只讲用户量不讲变现 |
| 6. 团队 | 为什么是你们做这件事（Founder-Market Fit） | 头衔堆砌，与业务无关 |
| 7. 财务预测 | 3 年 P&L、关键假设、里程碑 | 曲棍球杆式曲线无假设支撑 |
| 8. 融资计划 | 要多少钱、花在哪、花完达到什么状态 | "花完到达下一轮里程碑"缺失 |

**技巧**：投资人平均一份 BP 看 3 分钟——前 3 页（问题/方案/一句话模式）决定生死；财务预测部分务必附敏感性分析（见下节），这会让专业度立刻高一档。完整版式见 [商业计划书模板](../07-工具与模板库/模板/商业计划书模板.md)。

## 六、敏感性分析：给关键假设 ±20% 的压力测试

### 概念

任何测算（LTV、市场规模、财务预测）都建立在假设上。敏感性分析回答：**哪个假设错了最致命？** 方法是让关键假设逐一 ±20%，看结果的波动幅度。

### 操作步骤

1. 列出模型的 3-6 个关键输入假设（留存率、客单价、CAC、转化率、佣金率…）；
2. 固定其他变量，让每个假设单独 ±20%，记录输出（利润/NPV/LTV-CAC）的变化；
3. 按输出波动幅度排序 → 得到**敏感度排序**；
4. 对最敏感的 1-2 个假设：加密验证（做实验/深访），并准备预案。

### 示例：上文咖啡外送模型的敏感性

| 假设 ±20% | 成熟期月利润变化 | 敏感度 |
|-----------|----------------|--------|
| 月均购买频次（3 → 2.4/3.6） | −38% / +38% | ★★★ 最致命 |
| 单均收入（16 → 12.8/19.2） | −35% / +35% | ★★★ |
| 配送成本（3.5 → 4.2/2.8） | −22% / +22% | ★★ |
| CAC（40 → 48/32） | −12% / +12% | ★ |

**So What**：模型对"频次"最敏感——所以该业务的头号命题不是拉新（CAC 敏感度最低），而是复购运营；上线前必须用小范围实测验证频次假设，而不是继续优化投放。**敏感性分析的产出是行动优先级，不是表格本身。**

进阶：把最敏感的两个变量做成双变量矩阵（情景分析），标出"乐观/中性/悲观"三情景——BP 和立项书的标配。

## 七、技巧的组合使用：一个完整交付的样例流程

任务："一周内评估我们要不要跟进竞对刚上线的 X 业务。"

```
Day 1  竞对情报（技巧三）：财报电话会+JD+应用商店，还原对手的投入与预期
Day 2  快速算账（技巧一）：用可得数据搭 X 业务的单位经济草稿
Day 3  拆财报（技巧二）：对手的现金流撑得起这场仗吗？我们的呢？
Day 4  反共识检验（技巧四）：跟进=共识动作；有没有更好的非对称回应？
Day 5  敏感性分析（技巧六）：结论对哪个假设最敏感？该假设置信度多高？
产出   3 页决策备忘录（结构=BP 八段论的裁剪版）：结论先行 + 关键账 + 预案
```
